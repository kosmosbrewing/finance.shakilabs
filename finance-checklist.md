# shakilabs.com MVP 프로젝트 마무리 품질 체크리스트 v2

> 모든 서브도메인 프로젝트(finance, seller, biz 등)의 배포 전 최종 점검용.
> 각 항목: ✅ 통과 / ⚠️ 보류 / ❌ 미통과

---

## 1. 기능 정합성 (Functional Correctness)

### 1-1. 계산 로직 검증
- [ ] 핵심 계산 결과가 공식 출처(국세청, 4대보험공단 등)와 일치하는가
- [ ] 경계값 테스트: 최솟값, 최댓값, 0, 음수 입력 시 정상 처리되는가
- [ ] 소수점/원 단위 절사가 일관적인가 (Math.floor 통일)
- [ ] 기본값 로드 시 결과가 즉시 표시되는가 (Zero Friction 원칙)
- [ ] 빠른 선택 버튼 클릭 → 입력값 + 결과 동기화 정상인가
- [ ] 상세 설정(접힌 영역) 변경 시 결과가 즉시 재계산되는가

### 1-2. 라우팅 & 네비게이션
- [ ] 모든 URL 경로가 정상 접근 가능한가 (직접 입력, 새로고침)
- [ ] SEO 자동 생성 페이지(/:amount, /:a-vs-:b 등)가 올바르게 렌더링되는가
- [ ] 탭 네비게이션 활성 상태가 현재 경로와 일치하는가
- [ ] 내부 순환 링크 4개가 모두 정상 작동하는가
- [ ] 404 페이지가 존재하고 위트 있는 문구가 표시되는가

### 1-3. 공유 & 바이럴
- [ ] 카카오톡 공유 시 OG 카드(제목, 설명, 이미지)가 올바르게 표시되는가
- [ ] 링크 복사 → 붙여넣기 시 URL 파라미터로 상태가 복원되는가
- [ ] 공유 URL이 의미 있는 결과를 보여주는가 (빈 페이지가 아닌)

### 1-4. 커뮤니티 (Supabase)
- [ ] 댓글 작성/조회/좋아요가 정상 동작하는가
- [ ] 투표 기능이 정상 동작하고 결과가 반영되는가
- [ ] 300자 제한이 클라이언트 + Supabase RLS 양쪽에서 적용되는가
- [ ] XSS 방지: 댓글 내용이 이스케이프 처리되는가

## 2. SEO (검색 엔진 최적화)

### 2-1. 메타태그
- [ ] 각 페이지별 고유한 title 태그가 설정되는가 (60자 이내)
- [ ] 각 페이지별 고유한 meta description이 설정되는가 (155자 이내)
- [ ] OG 태그(og:title, og:description, og:image, og:url)가 페이지별로 동적 생성되는가
- [ ] canonical URL이 올바르게 설정되는가

### 2-2. 구조화 데이터
- [ ] FAQ 페이지에 JSON-LD FAQPage 스키마가 포함되는가
- [ ] JSON-LD가 Google 구조화 데이터 테스트 도구에서 오류 없는가

### 2-3. SPA 검색봇 대응 (크리티컬)
- [ ] vite-plugin-prerender 또는 vite-ssg로 정적 HTML이 사전 생성되는가
- [ ] `curl -A Googlebot` 으로 접근 시 메타태그 + 본문 콘텐츠가 HTML에 포함되는가
- [ ] Google Search Console → URL 검사 → 라이브 테스트에서 콘텐츠가 렌더링되는가
- [ ] prerender 대상 경로 목록이 SEO 페이지(약 60~70개)를 모두 포함하는가

### 2-4. 크롤링
- [ ] sitemap.xml에 모든 SEO 페이지가 포함되는가
- [ ] robots.txt가 올바르게 설정되는가 (크롤링 허용, 불필요 경로 차단)
- [ ] Google Search Console에 사이트맵 제출했는가

### 2-5. 내부 링크
- [ ] 4개 메인 페이지 간 순환 링크가 모두 연결되는가
- [ ] SEO 상세 페이지에서 메인 페이지로 돌아가는 링크가 있는가
- [ ] 앵커 텍스트가 타겟 키워드를 포함하는가

## 3. 성능 (Performance)

### 3-1. Core Web Vitals
측정 환경: PageSpeed Insights (모바일 시뮬레이션 기준), 배포 후 CDN 캐시 적용 상태에서 측정
- [ ] LCP (Largest Contentful Paint) ≤ 2.5초
- [ ] INP (Interaction to Next Paint) ≤ 200ms
- [ ] CLS (Cumulative Layout Shift) ≤ 0.1
- [ ] Lighthouse 성능 점수 ≥ 90 (PageSpeed Insights 모바일 기준)

### 3-2. 번들 & 로딩
- [ ] Vite 빌드 결과물 크기가 합리적인가 (JS < 200KB gzipped)
- [ ] 라우트별 코드 스플리팅이 적용되는가 (lazy import)
- [ ] 폰트(Pretendard, Inter)가 font-display: swap으로 로드되는가
- [ ] 이미지(OG 카드, 파비콘)가 최적화되어 있는가 (WebP, 적절한 크기)

### 3-3. 계산 성능
- [ ] 입력 디바운스(300ms)가 적용되어 있는가
- [ ] 대량 데이터 테이블(연봉별 비교표 등)이 렌더링 지연 없이 표시되는가
- [ ] Chart.js 도넛 차트가 즉시 렌더링되는가

## 4. 보안 (Security)

### 4-1. 입력 검증
- [ ] 모든 사용자 입력이 Zod 스키마로 검증되는가
- [ ] 숫자 입력에 음수, NaN, Infinity가 차단되는가

### 4-2. Supabase RLS 정책
- [ ] INSERT: anon 유저 댓글 작성 허용, rate limit 적용(분당 5건 이내)
- [ ] SELECT: 공개 댓글만 조회 가능한가 (삭제/신고된 댓글 제외)
- [ ] DELETE/UPDATE: 관리자 외 수정/삭제 불가 정책이 RLS에 반영되는가
- [ ] 댓글 본문 길이 제한이 RLS 또는 DB constraint로 강제되는가

### 4-3. 환경 변수
- [ ] Supabase URL/Key가 .env에만 존재하고 코드에 하드코딩되지 않았는가
- [ ] .env.example이 제공되는가
- [ ] .gitignore에 .env가 포함되는가
- [ ] VITE_SUPABASE_ANON_KEY는 anon key인가 (service key 노출 금지)

### 4-4. HTTP 헤더
- [ ] X-Content-Type-Options: nosniff
- [ ] X-Frame-Options: DENY
- [ ] X-XSS-Protection: 1; mode=block
- [ ] vercel.json에 보안 헤더가 포함되는가

### 4-5. XSS/Injection
- [ ] 댓글 내용 렌더링 시 v-text 또는 이스케이프 처리되는가 (v-html 금지)
- [ ] URL 파라미터를 DOM에 직접 삽입하지 않는가
- [ ] 카카오 SDK 등 외부 스크립트가 공식 CDN에서만 로드되는가

## 5. 수익화 (AdSense)

### 5-1. 승인 전 필수
- [ ] 개인정보처리방침(/privacy) 페이지가 존재하는가
- [ ] 이용약관 페이지가 존재하는가
- [ ] 사이트 안내/소개 페이지가 존재하는가
- [ ] 각 페이지에 충분한 고유 콘텐츠가 있는가 (최소 300단어 이상)
- [ ] 사이트가 최소 2주 이상 운영되었는가 (애드센스 심사 기간)
- [ ] ads.txt 파일이 /public에 존재하는가 (승인 후 추가도 가능하지만 미리 준비)

### 5-2. 광고 배치 & CLS
- [ ] AdSlot 컴포넌트에 고정 높이(min-height)가 예약되어 CLS를 유발하지 않는가
- [ ] 광고가 핵심 입력/결과 영역 사이에 과도하게 끼어들지 않는가
- [ ] 모바일 앵커 광고가 입력 영역이나 빠른 선택 버튼을 가리지 않는가
- [ ] 광고 슬롯 개수가 페이지당 3~4개 이내인가
- [ ] 광고 로드 실패 시에도 레이아웃이 깨지지 않는가

### 5-3. CPC 최적화
- [ ] 페이지 콘텐츠가 타겟 키워드(금융/세금/보험/수수료)를 자연스럽게 포함하는가
- [ ] FAQ 섹션이 검색 의도에 맞는 롱테일 키워드를 포함하는가
- [ ] 페이지 title, h1, meta description에 타겟 키워드가 포함되는가
- [ ] 스크롤 유도 콘텐츠(상세 내역, 비교표, 차트)가 충분해서 체류 시간이 확보되는가
- [ ] 결과 카드 직후 첫 광고 위치가 자연스러운가 (가장 높은 CTR 지점)

## 6. 반응형 & 크로스 브라우저 (UI/UX)

### 6-1. 반응형
- [ ] 모바일(375px)에서 모든 페이지가 정상 표시되는가
- [ ] 태블릿(768px)에서 레이아웃이 적절한가
- [ ] 데스크톱(1280px)에서 사이드 패널(커뮤니티)이 정상 표시되는가
- [ ] 숫자 입력 시 모바일 숫자 키패드(inputmode="numeric")가 표시되는가
- [ ] 빠른 선택 버튼이 모바일에서 터치 영역(44px+)이 충분한가

### 6-2. 다크모드
- [ ] 다크모드 토글이 정상 작동하는가
- [ ] 모든 텍스트/배경/카드/테이블이 다크모드에서 가독성이 있는가
- [ ] 차트(Chart.js)가 다크모드에서 올바르게 표시되는가
- [ ] 다크모드 설정 저장: CSS prefers-color-scheme 기본 + 토글 시 data-theme 속성으로 오버라이드, 쿠키 또는 URL 파라미터로 유지 (localStorage 사용 금지 — Vercel Edge 환경 호환)

### 6-3. 크로스 브라우저
- [ ] Safari(iOS): 숫자 입력 시 자동 확대가 발생하지 않는가 (input font-size ≥ 16px)
- [ ] Safari(iOS): date input 스타일이 커스텀 DatePicker와 충돌하지 않는가
- [ ] 카카오 인앱 브라우저: 공유 후 돌아온 페이지가 정상 렌더링되는가
- [ ] Samsung Internet: 기본 기능(계산, 탭 전환, 공유)이 정상 작동하는가
- [ ] Chrome/Firefox 최신 버전에서 정상 작동하는가

### 6-4. 디자인 일관성 (OTT 톤 유지)
- [ ] Primary 컬러가 indigo-600 계열로 통일되는가
- [ ] 카드 스타일(rounded-2xl, shadow-sm, border)이 OTT와 동일한가
- [ ] 테이블 스타일(stripe, hover, 헤더 sticky)이 OTT와 동일한가
- [ ] 폰트(Pretendard + Inter)가 올바르게 로드되는가
- [ ] 숫자가 tabular-nums로 정렬되는가
